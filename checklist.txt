# Moribund Flash Deck System Checklist

## Core Principle

```text
Many input formats.
One canonical deck format.
Two user paths.
One study engine.
```

The old checklist was mostly about the universal converter. The new version needs to include the product flow too: **Quick Study** and **Deck Library** should both exist. Your old “Choose a deck...” file-picker flow was good for immediate use, but the app also needs a managed library for saved decks, progress, metadata, and editing.

# 2. Architecture Rule

Do not let every screen parse every format.

Correct:

```text
External file
  ↓
Importer
  ↓
ImportDeckDraft or MflashDeck
  ↓
Study Screen or Deck Builder
```

Wrong:

```text
Study screen parses CSV.
Study screen parses Markdown.
Study screen parses APKG.
Study screen parses XML.
Everyone screams.
```

Use this division:

```text
Importers:
  understand messy file formats

Deck Builder:
  lets the user repair and improve imported decks

Study Screen:
  studies normalized decks only

mflash:
  stable canonical format
```

This keeps the study engine boring, which is good. Boring here means maintainable, not spiritually beige.

# 3. Canonical Formats

## `.mflash`

Use `.mflash` as the main deck format.

```text
[ ] Human-readable UTF-8 JSON.
[ ] Full Unicode support.
[ ] Supports CJK, Georgian, Cyrillic, Greek, IPA, emoji.
[ ] Supports deck-level language defaults.
[ ] Supports per-card language overrides.
[ ] Supports tags.
[ ] Supports examples.
[ ] Supports hyperlinks.
[ ] Supports media references.
[ ] Supports notes.
[ ] Supports future unknown fields.
```

## `.mpack`

Use `.mpack` for decks with bundled media.

Recommended structure:

```text
my-deck.mpack
├── deck.mflash
└── media/
    ├── cover.png
    ├── card-0001-audio.mp3
    └── card-0002-image.jpg
```

Checklist:

```text
[ ] Use ZIP for v1.
[ ] Require deck.mflash at package root.
[ ] Put media under media/.
[ ] Use only relative paths inside deck.mflash.
[ ] Reject absolute paths.
[ ] Reject ../ path traversal.
[ ] Sanitize filenames.
[ ] Rename duplicate files safely.
[ ] Update media references inside deck.mflash.
[ ] Allow users to unzip and inspect manually.
```

# 4. Import Model

Add an intermediate model before saving to `.mflash`.

```text
[ ] Create ImportDeckDraft.
[ ] Create ImportCardDraft.
[ ] Importers return ImportDeckDraft.
[ ] ImportDeckDraft converts to MflashDeck.
[ ] ImportDeckDraft converts to frontend Deck Builder state.
[ ] Preserve warnings.
[ ] Preserve unknown source fields.
```

Suggested shape:

```rust
pub struct ImportDeckDraft {
    pub title: String,
    pub description: Option<String>,
    pub snippet: Option<String>,
    pub default_term_lang: Option<String>,
    pub default_def_lang: Option<String>,
    pub deck_tags: Vec<String>,
    pub cover_media: Option<String>,
    pub cards: Vec<ImportCardDraft>,
    pub source_format: String,
    pub warnings: Vec<String>,
}

pub struct ImportCardDraft {
    pub term: String,
    pub definition: String,
    pub term_lang: Option<String>,
    pub def_lang: Option<String>,
    pub hyperlink: Option<String>,
    pub media: Option<String>,
    pub tags: Vec<String>,
    pub examples: Vec<String>,
    pub notes: Option<String>,
    pub source_id: Option<String>,
    pub extra_fields: serde_json::Value,
}
```

Why: the old basic `Deck` model loses metadata. The import draft gives you a safe staging area.

# 5. Importer Checklist

## General Importer Rules

```text
[ ] Return ImportDeckDraft, not low-detail Deck.
[ ] Fatal error only when no usable cards exist.
[ ] Preserve Unicode exactly.
[ ] Preserve unknown metadata in extra_fields.
[ ] Return warnings for partial problems.
[ ] Do not silently discard useful fields.
[ ] Do not silently guess important metadata without user confirmation.
```

---

## TXT Importer

TXT is lossy, keep it simple.

```text
[ ] Keep newline-separated cards.
[ ] Keep semicolon-separated cards if useful.
[ ] Support tab separator.
[ ] Support " - " separator.
[ ] Support comma separator.
[ ] Do not use bare hyphen as a separator.
[ ] Prompt for deck title if missing.
[ ] Prompt for default term language.
[ ] Prompt for default definition language.
[ ] Warn when many definitions are empty.
[ ] Return ImportDeckDraft.
```

---

## CSV Importer

CSV should become the practical “spreadsheet import” route.

```text
[ ] Detect headers.
[ ] Support no-header CSV.
[ ] Support quoted commas.
[ ] Support flexible row lengths.
[ ] Detect term/front/question/word/prompt columns.
[ ] Detect definition/back/answer/meaning/translation columns.
[ ] Detect tags column.
[ ] Detect examples column.
[ ] Detect term_lang column.
[ ] Detect def_lang column.
[ ] Detect hyperlink/link/url column.
[ ] Detect media/image/audio/file column.
[ ] Detect notes column.
[ ] Preserve unknown columns in extra_fields.
[ ] Prompt for default languages if missing.
[ ] Return ImportDeckDraft.
```

---

## Markdown Importer

Markdown can be the “human writer’s format.”

```text
[ ] Keep heading-pair support.
[ ] Keep bullet-list support.
[ ] Keep table support.
[ ] Keep fenced card block support.
[ ] Keep glossary style support.
[ ] Keep term: definition line support.
[ ] Add YAML front matter.
[ ] Add richer fenced card blocks.
[ ] Preserve tags.
[ ] Preserve examples.
[ ] Preserve media.

[ ] Preserve notes.
```

```text
  ↓

  ↓

  ↓
TXT importer
```

Upgrade checklist:

```text
[ ] Keep ZIP extraction.
[ ] Keep SQLite reading.
[ ] Read notes table directly.
[ ] Read note IDs.
[ ] Preserve note ID as source_id.
[ ] Read note tags.
[ ] Read model field names.
[ ] Read deck names where practical.
[ ] Preserve extra Anki fields in extra_fields.
[ ] Detect [sound:...] references.
[ ] Detect image references.
[ ] Read APKG media map.
[ ] Copy media into mpack media folder later.
[ ] Stop converting APKG to synthetic TXT.
[ ] Return ImportDeckDraft.
```

APKG is where `.mpack` starts to matter.

# 15. Error and Warning Strategy

Importers should not just return success/failure. They should return useful warnings.

Checklist:

```text
[ ] Warn when definitions are missing.

[ ] Warn when language guess is uncertain.
[ ] Warn when APKG fields were preserved in extra_fields.
```

```text

8 media references could not be found.
Language guessed as Japanese, please confirm.
```

# 17. Final Decision Rules

When unsure, use these:

```text
If the user wants to study right now:
  Study a Deck...

If the user wants a stable saved collection:
  Deck Library

If the user wants to create or repair a deck:
  Deck Builder

If the file is messy:
  ImportDeckDraft

If the file is clean and canonical:
  MflashDeck

If the deck has media:
  mpack

If the study screen starts parsing CSV:
  Stop. The raccoon has entered the circuitry.
```

This version is the stronger plan: it keeps the old file-picker convenience, adds a real library model, protects the study screen from importer chaos, and makes `.mflash` the center of the system.
